Open-source video rendering platform built for AI agents. Define compositions in plain HTML, render deterministically to MP4 — no timeline editor, no proprietary formats. Open-source платформа рендеринга видео для AI агентов. Пиши compositions на обычном HTML, рендери детерминированно в MP4 — без редактора таймлайна и проприетарных форматов.

Get Started Free →Начать бесплатно → See how it worksКак это работает
compositions/launch.html → out.mp4 
<!-- Your video is just HTML -->
<div data-composition-id="launch"
     data-width="1920" data-height="1080">

  <video id="bg" class="clip"
         data-start="0" data-duration="30" data-track-index="0"
         src="assets/bg.mp4" muted playsinline/>

  <h1 id="title" class="clip"
       data-start="1.2" data-duration="8" data-track-index="1"
       style="font-size:96px; color:#d4f542">
    Product Launch 2026
  </h1>

</div>

# Then just render
$ npx reframemotion render --quality high --output launch.mp4
✓ 900 frames → launch.mp4 (30s @ 30fps)

How it worksКак работает

From brief to MP4
in three stepsОт брифа до MP4
за три шага

01

Describe — agent writes HTMLОпиши — агент пишет HTML

Prompt Claude Code, Cursor, or any agent. The /hyperframes skill teaches it exact composition patterns, GSAP animation rules, and timing constraints.Промпти Claude Code, Cursor или любого агента. Skill /hyperframes обучает его точным паттернам compositions, правилам GSAP и ограничениям тайминга.

Using /hyperframes, create a 30s
product demo with GSAP animations
and TTS narration synced to captions.
02

Preview — instant feedbackПревью — мгновенная обратная связь

Live Studio in the browser. Edit HTML, see changes instantly. Lint catches timing conflicts. WCAG contrast validator built-in.Живой Studio в браузере. Редактируй HTML, видь изменения сразу. Lint ловит конфликты тайминга.

$ npx reframemotion preview
Studio → localhost:3000
Hot reload on every save
03

Render — deterministic MP4Рендер — детерминированный MP4

Chrome BeginFrame API captures each frame. FFmpeg encodes to MP4/WebM. GPU-accelerated. Parallel workers auto-scale by CPU/RAM.Chrome BeginFrame API захватывает каждый кадр. FFmpeg кодирует в MP4/WebM. GPU-ускорение. Параллельные workers автомасштабируются.

$ npx reframemotion render
900 frames captured in parallel
✓ output.mp4 ready in 38s
