# Wiki по программному созданию видео

**Версия исследования:** 2026-07-16  
**Основные движки:** HyperFrames, Remotion, FFmpeg  
**Интеграции:** MCP, Agent Skills, Codex/Claude/Cursor, очереди и облачный рендер

Эта wiki — практическая база для разработки видео кодом, генераторов видео и агентных производственных пайплайнов. Она не является копией документации: здесь собраны архитектурные решения, ограничения, типовые ошибки, чек-листы, шаблоны и критерии выбора.

## Быстрый вывод

- **HyperFrames** рационален, когда основа — HTML/CSS/GSAP, важны захват сайта, свободная DOM-композиция, агентная генерация и открытая лицензия Apache 2.0.
- **Remotion** рационален, когда команда уже работает на React/TypeScript, нужен встроенный Player, типизированные шаблоны, интерактивный редактор или зрелый AWS Lambda-рендер.
- **FFmpeg** следует использовать отдельно для склейки, транскодирования, нормализации, извлечения кадров, аудиоанализа и других задач, где браузерная композиция не нужна.
- В production обычно выигрывает **гибрид**: данные и медиа → FFmpeg/анализ → HyperFrames или Remotion → проверка → очередь → рендер → хранилище/CDN.

## Рекомендуемый порядок для первого проекта

1. Заполнить `VIDEO-BRIEF`.
2. Выбрать движок по `01-stack-selection.md`.
3. Зафиксировать визуальную систему в `DESIGN.md`.
4. Написать сценарий и storyboard до кода.
5. Сначала собрать статические hero-кадры сцен.
6. Добавить анимацию, звук и переходы.
7. Выполнить автоматические и визуальные проверки.
8. Рендерить черновик, затем финал в закрепленном окружении.
9. Сохранить входные данные, версии зависимостей и manifest результата.

## Главный принцип

Видео, сгенерированное кодом, должно быть **воспроизводимым продуктом**, а не одноразовым эффектом. Поэтому source of truth — не MP4, а совокупность:

- входных данных;
- замороженных ассетов;
- дизайн-токенов;
- сценария и таймингов;
- исходного кода композиции;
- версий движка, браузера и FFmpeg;
- журнала проверок и параметров рендера.
