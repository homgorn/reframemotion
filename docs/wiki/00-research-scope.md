# 00. Область исследования и надежность

## Что исследовано

Wiki основана на четырех слоях источников:

1. Встроенные навыки HyperFrames и Remotion, подключенные к агенту.
2. Актуальная официальная документация и репозитории на 16 июля 2026 года.
3. Официальные showcase/success stories и открытые примеры.
4. Архитектурные выводы, выведенные из общих принципов browser rendering, React, DOM, FFmpeg, облачных очередей и MCP.

## Что здесь считается фактом

**Факт** — утверждение, непосредственно подтвержденное официальной документацией, исходным кодом или спецификацией. Например:

- HyperFrames использует HTML как источник композиции и управляемый покадровый захват браузера.
- Remotion вычисляет визуальное состояние по кадру и рендерит React-композиции.
- MCP разделяет data layer и transport layer и использует JSON-RPC 2.0.
- Remotion Cloud Run на дату исследования помечен как Alpha и не развивается активно.

## Что считается интерпретацией

**Интерпретация** — инженерный вывод с указанием условий. Например:

- HyperFrames обычно проще для агентной генерации нестандартных DOM-сцен.
- Remotion обычно удобнее React-команде для построения редактора с live preview.

Эти выводы не универсальны: квалификация команды, инфраструктура, лицензия и существующий код могут изменить выбор.

## Ограничения исследования

- Невозможно перечислить буквально все существующие кейсы: их число неограниченно. Вместо этого дана полная типология и репрезентативные реализации.
- Официальные success stories — маркетинговые материалы. Они подтверждают способ применения, но финансовые показатели компаний следует считать self-reported, если нет независимого аудита.
- HyperFrames развивается быстро. Встроенные навыки и текущая документация местами отражают разные версии поведения. В спорных местах wiki рекомендует закрепить версию и проверить результат CLI-командами.
- Тарифы, лицензии, облачные лимиты и команды CLI могут измениться после даты исследования.

## Версионная дисциплина

Для production-проекта сохранять:

```text
Node/Bun version
hyperframes or remotion exact version
GSAP version (HyperFrames)
Chrome/Chromium revision
FFmpeg version
OS/container image digest
font file checksums
asset checksums
render command and flags
input props / variables JSON
```

Использовать lock-файл и запрещать неявное обновление major/minor-версий в CI.

## Уровни уверенности

- **Высокая:** базовая архитектура обоих движков, покадровая детерминированность, правила media playback, текущие официальные API.
- **Средняя:** сравнительная производительность и удобство агентной генерации — зависят от конкретной композиции и команды.
- **Низкая:** любые общие заявления вида «движок X всегда быстрее/лучше/креативнее» без воспроизводимого benchmark.
